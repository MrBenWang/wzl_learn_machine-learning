问题1 - 特征观察¶

如前文所述，本项目中我们关注的是其中三个值:'RM'、'LSTAT' 和'PTRATIO'，对每一个数据点:

  • 'RM' 是该地区中每个房屋的平均房间数量；
  • 'LSTAT' 是指该地区有多少百分比的房东属于是低收入阶层（有工作但收入微薄）；
  • 'PTRATIO' 是该地区的中学和小学里，学生和老师的数目比（学生/老师）。

凭直觉，上述三个特征中对每一个来说，你认为增大该特征的数值，'MEDV'的值会是增大还是减小呢？每一个答案都需要你给出理由。

提示：你预期一个'RM' 值是6的房屋跟'RM' 值是7的房屋相比，价值更高还是更低呢？

回答:

  • RM 增大时，MEDV 会增大。因为，房间数量变多，建造的成本也会增加，价格自然会上升。
  • LSTAT 增大时，MEDV 会减小。因为，房东是低收入阶层增多时，房屋的价格必然是处于比较低的位置。
  • PTRATIO 增大时，MEDV 会减小。因为，一个老师对应更多的学生，意味着周围的家庭数量更多，此时的房屋供应的体量也很大，所以价格很难上升。

练习：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。_模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

def performance_metric(y_true, y_predict):
    """ Calculates and returns the performance score between 
        true and predicted values based on the metric chosen. """
    
    # TODO: Calculate the performance score between 'y_true' and 'y_predict'
    score = r2_score(y_true, y_predict) 
    
    # Return the score
    return score

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.grid_search import GridSearchCV 

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = range(1, 11)
    params = {"max_depth": params}
    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search object
    grid = GridSearchCV(regressor, params, scoring_fnc, cv = cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

    • 客户2，他的房间数比客户1少一间房间，价格略低于客户1，可以忽略这个特征；社区平困指数比客户1大一倍左右，价格应该是客户1价格的一半左右；学生/老师比例比客户1高出 68%，价格应该是客户1价格的四分之三左右。综合三个特征和客户2的价格，得出客户2是客户1近一半左右的价格在合理预测范围内。
    • 客户3，他的房间数量比客户1多出一倍左右，价格应该是客户1的价格的两倍，社区平困指数远低于客户1，价格应该是客户1的两倍往上，学生/老师比例比客户1略低，价格也应比客户1略高一些，可以忽略这个特征。综合三个特征和客户3的价格，得出客户3是客户1大于2倍左右的价格在合理预测范围内。

问题 11 - 实用性探讨¶

简单地讨论一下你建构的模型能否在现实世界中使用？

提示： 回答几个问题：

  • 1978年所采集的数据，在今天是否仍然适用？
  • 数据中呈现的特征是否足够描述一个房屋？
  • 模型是否足够健壮来保证预测的一致性？
  • 在波士顿这样的大都市采集的数据，能否应用在其它乡镇地区？

答案:

构建的模型可以在现实世界中使用，只能作为一个辅助参考的手段。它能够结合已有的特征和价格的之间的关系，对未知的数据进行相对较为合理的预测。但是由于数据量较小，数据的时间跨度较大，特征不够充分，因素区域，以及一些其他的未知因素等。不能够泛化到所有的房价预测模型上。具体的因素分析如下：

  • 1978年的数据 太过久远，今天不适用；
  • 数据中只有三个特征，不够描述一个房屋，比如可能还包括距离市中心的距离，向阳方向等；
  • 足够健壮，可以保持预测的一致性；
  • 不可以应用在其他乡镇地区，因为这里可能还有其他的内在关系没有被发掘。
